③ Zustand Store Connections
chats.store persisted
activeTab — "chats" | "channels" | "groups"
activeChatId — currently open chat ID
directChats[] — DM list items (id, name, userId, lastMsg, time)
channels[] — sparse channel list from store
groups[] — sparse group list from store
messages — Record<chatId, ChatMessage[]>
replyingTo — { id, text, senderName } | null
unreadCounts — Record<chatId, number>
mutedChatIds[] — muted DM IDs
isComposing — compose DM open state
communities.store persisted
myChannels[] — full CommunityChannel objects (has desc, avatarUrl, links…)
myGroups[] — full CommunityGroup objects (has desc, activity, members…)
channelBroadcasts — Record<chanId, Broadcast[]>
groupMessages — Record<groupId, GroupMessage[]>
pinnedBroadcastIds — Record<chanId, string>
pinnedGroupMessageIds — Record<groupId, string>
mutedChannelIds[]
mutedGroupIds[]
joinedGroupIds[]
subscribedChannelIds[]
users.store seed only
users[] — 12 seed users
avatarUrl — ui-avatars.com URL
bio — user bio string
isOnline — boolean
lastSeen — ISO timestamp
getUser(id) — lookup by ID
getUserByName(name) — lookup by name
formatLastSeen(user) — "Active 5m ago"
auth.store persisted
user.id — used to determine isOwner
user.name — shown as "You" in messages

privacy.store persisted
addChannel() — called on block user action
blockedChannels[] — blocked user/channel IDs
④ Data Resolution Flow — how activeData is built
Active Tab Data Source Priority Why Fields gained from communitiesStore
chats directChats.find() DMs only exist in chats.store avatarUrl + bio resolved separately via users.store using userId or getUserByName()
channels [...myChannels, ...channels].find()
communitiesStore first		 myChannels has rich data (desc, avatarUrl, links, members). chats.store channels are sparse stubs. desc, avatarUrl, links[], members, isPrivate, category
groups [...myGroups, ...groups].find()
communitiesStore first		 myGroups has rich data (desc, activity, groupMembers). chats.store groups are sparse stubs. desc, avatarUrl, activity, groupMembers[], admins[]
⑤ User Interaction Flows
Open a Chat / Channel / Group
1
User clicks a list item in DirectChatList / ChannelList / GroupList
↓
2
setActiveChatId(id) called on chats.store → auto-calls markRead(id) → clears unread badge
↓
3
page.tsx re-derives activeData by searching correct store array
↓
4
Matching feed renders: header, messages/broadcasts, input bar (if not channel)
↓
5
Feed auto-scrolls to bottom via scrollRef.scrollTop = scrollHeight
Send a Message
1
User types in ChatInput textarea
↓
2
Press Enter or click Send → sendMessage(text) called on chats.store
↓
3
If replyingTo is set, message includes replyTo object. setReplyingTo(null) clears bar.
↓
4
New message appended to messages[activeChatId], status = "sent"
↓
5
GroupContentFeed detects new message by isMe, triggers typing indicator for 1.8s
Open Info Panel
1
User clicks ℹ button in feed header → handleToggleInfo() in page.tsx
↓
2
showInfo = true → ChatBoxSize renders info panel div as absolute overlay
↓
3
renderInfoPanel() picks correct panel based on activeTab
↓
4
Panel resolves bio from users.store (DM) or communitiesStore (channel/group)
↓
5
Panel slides in from right with animate-in slide-in-from-right-8. Content never resizes.
React to a Message
1
User hovers ChatBubble → action bar appears (Smile, Reply, Copy, Delete)
↓
2
Click Smile → 6-emoji picker popup appears above the bubble
↓
3
Pick emoji → onReact(msgId, emoji) fires:
DM → addReaction() on chats.store
Group → addGroupMessageReaction() on communitiesStore
↓
4
Toggle logic: if userId already in array → remove, else → add. Reaction pill count updates.
Switch Tabs
1
User clicks Chats / Channels / Groups in DisplayFeature tab bar
↓
2
setActiveTab(tab) updates chats.store. activeChatId is NOT reset.
↓
3
Sidebar renders matching list. Content renders matching feed.
↓
4
showInfo stays as-is (ℹ panel may briefly show wrong type until user selects new item)
Compose New DM
1
User clicks ✏️ pencil button in sidebar header
↓
2
setIsComposing(true) → compose input slides down from header
↓
3
User types name → press Enter → addDirectChat(name) on chats.store
↓
4
New DM entry appended to directChats[] with generated ID, auto-selected

→ "Select a conversation"
→ click item →
activeChatId = "123"
→ feed + input renders
→ switch tab →
activeChatId preserved
→ may show wrong feed
Info Panel State
showInfo = false
→ no overlay
→ click ℹ →
showInfo = true
→ overlay slides in
→ click item in sidebar →
showInfo = false
→ auto-closed
Reply State (ChatInput)
replyingTo = null
→ normal input
→ click Reply on bubble →
replyingTo = {"{id, text, senderName}"}
→ reply bar shown, placeholder changes
→ send or Esc →
replyingTo = null
